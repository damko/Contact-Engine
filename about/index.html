What is Contact Engine

It’s a middleware service meant to handle via API-REST information about People, Organizations, Locations and the relationships between the three objects.

Technical details

Contact Engine is a Code Igniter application (with Spark) loaded into RestIgniter.

The programming language is of PHP and the backend is LDAP and natively supports master-slave LDAP infrastructures. How it works

Contact Engine acts like the opposite of a common ORM: the properties (attributes) of the 3 objects are automatically set by parsing the ldap schema. For this reason, adding a new attribute to the schema for one o more objects is enough to add a new attribute to the object. You can refine the attributes of the 3 objects as you like without rewriting a single line of code. Architecture

The architecture of CE allows data access through 2 possible path: via LDAP and via REST as shown in the picture below.

Advantages

Because CE’s architecture, it has some unique advantages: * Any web application can connect to CE, doesn’t matter the programming language used. * One centralized repository for all your contacts * Programmers do not reinvent the wheel: all yours applications have a unique way to access contacts data and you can forget about it. * Taking the advantage of the community: updates of the code are immediately released and everybody can benefit. * Opensourse code released under GPL: in case of need you can modify the code as you like. * You own your data. Your data are on your server in your ldap. * Funambol ready: All the contacts stored by CE can be handled (one way) by a Funambol server without writing a single line of code. In this way contacts data can be kept synchronized on mobile phones and desktop applications even if they are not ldap-aware or rest-aware (they still funambol connector though, but there is large support for devices). * Strong and mature software: CE is new and still in development but is based on solid and well tested software.
